# Person-Detector

Детектор людей на основе архитектуры YOLO. Обучение с нуля на датасете COCO (класс "person").

### Выбор датасета

Для обучения детектора людей выбран **COCO** как наиболее сбалансированный датасет, обеспечивающий:

- ✅ **Большой объем** — >200,000 изображений, ~250,000 аннотаций людей
- ✅ **Разнообразие** — различные сцены, ракурсы, условия освещения
- ✅ **Наличие атрибутов** — разметка позы, окклюзии
- ✅ **Статус бенчмарка** — возможность сравнения результатов с другими работами

Для оценки устойчивости модели к перекрытиям дополнительно будет проведено тестирование на **CrowdHuman**.

## Быстрый старт

1. Клонировать репозиторий
2. Создать виртуальное окружение
3. Установить зависимости: `pip install -r requirements.txt`
4. Инициализировать и скачать подмодуль YOLOv5 `git submodule update --init --recursive`
5. Скачать COCO в папке yolov5: `cd yolov5 bash data/scripts/get_coco.sh`
6. Протестировать работу на coco128 с предобученной моделью в папке yolov5:
   `python detect.py --source data/images/bus.jpg --weights yolov5s.pt`
7. Запускаешь `python src/filter.py`, что бы создать датасет только с person
8. После запуска появятся файлы:

- train_person_split.txt — для обучения
- val_person.txt — для валидации
- test_person.txt — для теста
- А в папке datasets/coco/labels_person/ — отфильтрованные YOLO-лейблы (только класс 0).

9. Важно: После фильтрации нужно переименовать папки, чтобы YOLO использовал отфильтрованные лейблы:
   `cd datasets/coco`
   `mv labels labels_backup`
   `mv labels_person labels`
   `cd ../..`
10. Модель готова к обучению!

### MaxVP

В идеальном сценарии круто было бы использовать подход, предложенный в работах Unity Technologies: **предобучение на синтетическом датасете PeopleSansPeople** с последующей донастройкой на реальных данных.

**План MaxVP:**
1. **Предобучение на PeopleSansPeople** (500,000+ изображений, 3,000,000+ аннотаций)
2. **Донастройка на COCO-Person** (64,115 изображений)
3. **Тестирование на CrowdHuman** (экстремальные перекрытия)

**Ожидаемые преимущества:**
- ✅ Улучшение обобщающей способности модели
- ✅ Повышение устойчивости к окклюзиям и сложным ракурсам
- ✅ Снижение влияния шума в данных за счет идеальной разметки

**Архитектура лучшая для такой задачи:**

В качестве альтернативы YOLO 
была рассмотрена архитектура **DETR** (Detection Transformer), 
предложенная в работе *"End-to-End Object Detection with Transformers"* 
(Carion et al., 2020). В отличие от традиционных CNN-детекторов, DETR 
использует механизм внимания (transformer) для анализа связей между всеми частями изображения.

**Преимущества DETR:**
- **Глобальный контекст** — модель учитывает связи между всеми объектами на изображении, что особенно важно в сценах с сильными перекрытиями
- **Отказ от NMS** — предсказания выполняются как задача set prediction, что упрощает пайплайн
- **Эффективность в сложных сценах** — показывает лучшие результаты на датасетах с высокой плотностью объектов (CrowdHuman)

**Недостатки DETR:**
- **Более медленное обучение** — требует больше эпох для сходимости (до 500 эпох против 100 у YOLO)
- **Высокие требования к ресурсам** — для обучения требуется значительно больше видеопамяти
- **Сложность реализации с нуля** — требует глубокого понимания трансформеров и механизмов внимания

**Почему выбран YOLO:** Для решения задачи детекции людей в рамках 
ограниченного времени и мощности YOLO обеспечивает оптимальный 
баланс между скоростью разработки, вычислительными затратами и качеством результата.
DETR был рассмотрен как перспективное направление, но отложен из-за высоких 
требований к ресурсам и времени обучения.

### Результаты тестирования

Модель YOLOv5s была обучена с нуля на датасете COCO (класс "person") в течение 96 эпох. Обучение заняло 8.6 часов на видеокарте NVIDIA RTX 4070 (12 ГБ).

### Финальные метрики

| Выборка | Изображений | Инстансов | Precision | Recall | mAP@0.5 | mAP@0.5:0.95 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Валидация** (val2017) | 2,693 | 10,777 | **0.822** | **0.688** | **0.793** | **0.532** |
